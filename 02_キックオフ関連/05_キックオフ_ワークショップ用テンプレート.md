# キックオフ ワークショップ用テンプレート

## 🎯 ワークショップの目的

キックオフミーティングで、業務チームから「よく来る質問」と「対応が大変な業務」を洗い出し、AIエージェントで最初に取り組むべきトピックを決定します。

**所要時間:** 約20分

---

## 📝 ワーク1: 「またこの質問か…」リスト（10分）

### 進め方

#### ステップ1: 個人で付箋に書き出す（5分）

以下の質問を参考に、**付箋1枚につき1つの質問**を書いてください。

**質問:**
- 「またこの質問か…」とため息が出るものは？
- 新入社員や中途入社者から必ず聞かれることは？
- マニュアルを見れば書いてあるのに、なぜか電話がかかってくる件は？

**書き方の例:**
```
┌─────────────────┐
│ 有給休暇の     │
│ 申請方法       │
│               │
│ （経験: 週3回） │
└─────────────────┘
```

**ポイント:**
- できるだけ具体的に書く
- 頻度がわかれば書き添える（例: 「週に3回」「毎月」）
- 思いつく限り書く（最低3枚、上限なし）

---

#### ステップ2: ホワイトボードに貼る（2分）

書いた付箋を前のホワイトボードに貼ってください。

---

#### ステップ3: グルーピング（3分）

ファシリテーターが似たような質問をまとめます。

**グルーピング例:**
```
┌────────────────────┐
│【経費精算】        │
│ ・交通費申請       │
│ ・領収書の扱い     │
│ ・定期区間外の精算 │
└────────────────────┘

┌────────────────────┐
│【勤怠】            │
│ ・有給申請         │
│ ・振替休日         │
│ ・遅刻・早退       │
└────────────────────┘
```

---

## 🗳️ ワーク2: 優先順位の投票（5分）

### 進め方

#### ステップ1: 投票ルールの説明（1分）

各自に**3票**を配ります（シールや丸印）。

**投票基準:**
- 問い合わせ頻度が高い
- 対応に時間がかかる
- AIで解決できそう

---

#### ステップ2: 投票（2分）

ホワイトボードのグループに投票してください。
- 1つのグループに3票すべて入れてもOK
- 分散して投票してもOK

---

#### ステップ3: 集計・発表（2分）

票数の多い**Top 3**を発表します。

**結果記入欄:**

| 順位 | カテゴリ/トピック | 票数 |
|:---|:---|---:|
| 1位 | | 〇票 |
| 2位 | | 〇票 |
| 3位 | | 〇票 |

---

## 📋 ワーク3: トピック定義シートの記入説明（5分）

### 進め方

Top 3に決まったトピックについて、次回までに「トピック定義シート」を記入してもらいます。

#### 記入例を見せる

```
例: 「交通費申請」が1位になった場合

【①業務カテゴリ】
経費精算

【②ユーザーの質問（トリガー）】
・交通費を申請したい
・電車代の請求方法は？
・定期区間外の移動費
・Suicaの履歴で申請できる？

【③確認事項（条件分岐の基準）】
・それは「通勤」ですか？「出張」ですか？
・領収書はありますか？

【④回答内容・案内先】
・通勤の場合 → Aシステムへ
・出張の場合 → Bシステムへ（領収書添付必須）

【⑤参照すべきナレッジ（根拠）】
旅費規程 第3条
経費精算マニュアルP.10
```

---

## 🎨 ファシリテーター向けTips

### 準備物

- [ ] 付箋（1人あたり10枚程度）
- [ ] ペン（太めのマーカー推奨）
- [ ] ホワイトボード or 模造紙
- [ ] 投票用シール（1人3枚）or マーカー

---

### 進行のポイント

#### ✅ やるべきこと

**1. 具体例を最初に示す**
```
「例えば、私が担当部門に問い合わせたことで、
『有給の申請方法』とか『交通費の精算』とかありますよね。
そういうものを思いつく限り書いてください」
```

**2. 沈黙を恐れない**
「個人ワーク中は静かになりますが、それでOKです。5分間、集中して書き出してください」

**3. 数を出すことを優先**
「完璧でなくても大丈夫です。とにかく数を出しましょう」

---

#### ❌ 避けるべきこと

**1. 最初から絞り込もうとする**
「これは対象外」「これはAIには無理」などと最初から除外しない

**2. 議論を長引かせる**
グルーピングで意見が割れたら「仮でここに入れておきます」と決める

**3. 完璧を求める**
「詳細は後で詰めます。今日はざっくりで大丈夫」

ワークショップ終了後、以下を撮影してください:

- [ ] ホワイトボード全体（付箋が貼られた状態）
- [ ] グルーピング後の状態
- [ ] 投票結果（票数がわかる状態）

---

## 🔄 時間が押した場合の調整

### パターンA: 個人ワークが長引いている（5分→7分）
→ グルーピングを簡略化（似たものだけざっくりまとめる）

### パターンB: グルーピングで議論が長引く
→ 「仮置き」として進め、後で調整

### パターンC: 全体が押している
→ 投票を省略し、ファシリテーターが票数の多そうなものをTop 3として提案
