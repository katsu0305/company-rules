## 🚀 プロジェクトの進め方

### Phase 1: 準備・トピック設計（1ヶ月目）

#### Week 1: キックオフ
- [ ] [キックオフ資料.md](./02_キックオフ関連/03_キックオフ資料.md) を使ってキックオフミーティング開催
- [ ] プロジェクトの目的と役割分担を合意
- [ ] 最初に取り組むトピック（Top 3）を決定

#### Week 2-3: 情報収集
- [ ] 業務チームが [トピック定義シート.csv](./03_実践・運用フェーズ/07_トピック定義シート.csv) を記入
- [ ] [トピック定義シート_記入ガイド.md](./03_実践・運用フェーズ/08_トピック定義シート_記入ガイド.md) を参照しながら記入
- [ ] 既存FAQのリストアップ

#### Week 4: ナレッジ整理
- [ ] [ナレッジ整理チェックリスト.md](./03_実践・運用フェーズ/09_ナレッジ整理チェックリスト.md) を使ってナレッジの棚卸し
- [ ] 正式版の決定、古い情報の削除

---

### Phase 2: 実装・試験運用（2ヶ月目）

#### Week 5-6: 実装
- [ ] 開発チームがトピック定義シートをもとに実装
- [ ] [KPI測定ダッシュボード設計書.md](./03_実践・運用フェーズ/11_KPI測定ダッシュボード設計書.md) をもとにPower BI構築

#### Week 7-8: 試験運用
- [ ] 限定ユーザーでテスト開始
- [ ] [定例会運用ガイド.md](./03_実践・運用フェーズ/10_定例会運用ガイド.md) に沿って週次レビュー会を開始

---

### Phase 3: 本格運用・改善（3ヶ月目〜）

#### Week 9以降: 継続改善
- [ ] 全社展開
- [ ] 週次レビュー会の継続
- [ ] KPIモニタリング
- [ ] ナレッジの追加・更新
- [ ] [AI利用促進TIPS.md](./04t_tips/12_AI利用促進TIPS.md) を参考に利用促進施策を実施

---

### Phase 4: 他部門への横展開(4ヶ月目〜)

#### 横展開準備
- [ ] [部門展開_シチズンデベロッパー育成ガイド.md](./04t_tips/13_部門展開_シチズンデベロッパー育成ガイド.md) に基づき候補者選出
- [ ] [全社展開時のロール定義とRACI.md](./04t_tips/14_全社展開時のロール定義とRACI.md) でロール分担を明確化
- [ ] 部門開発者向けトレーニング実施(3日間)
- [ ] 先行部門のナレッジを活用した横展開
